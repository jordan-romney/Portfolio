Inventory Management App

View the Full Code Here

This Inventory Management App was a project from my Software Development Degree. This project highlights various Java concepts including:

  • Classes, objects, and methods
  • Inheritance, encapsulation, and abstraction
  • Exception Handling
  • User Interface Development
❮ ❯

This was my first major Java project. Each screen had its own FXML file and a controller. The project consists of several classes including an abstract class. Two classes inherit from the abstract to use common methods. The abstract Part class was created to allow for other classes to function without knowing the bigger picture. The attributes of a part could be utilized by more specific and specialized types of parts.

All the classes contain encapsulated private variables and methods that are controlled by “getter” and “setter” methods. This allows users to access and set some data without causing changes to more sensitive properties of an object.

Throughout the app, there are checks to ensure that the program can handle unwanted behavior. The main use of this exception handling is when a user adds or updates a part or product. The data types of each text field are checked against what is required to complete a part or product object. If the data type is incorrect an error dialogue is displayed, and the user can not proceed without fixing the error. This prevents the application from displaying any unwanted behavior.

This project was completed using JavaFX and SceneBuilder to create a Graphical User Interface. This was the first project I had made with a GUI and I enjoyed being able to visualize exactly how users would interact with my code. I feel that creating a GUI allowed me to write cleaner more effective code since I knew exactly how each method and class would be manipulated by the user.
