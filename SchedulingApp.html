Scheduling App

View the Full Code Here

This Scheduling App was a project from my Software Development Degree. This project highlights various Java concepts including:

  • Database and file server applications
  • Writing to an external file
  • Lambda expressions
  • Localization API and Date/Time API
  • Advanced exception control
❮ ❯

This was the largest Java project I have created to date and showed me the importance of organization. One of the more useful approaches I used was to create an abstract class of “helper” functions. These were functions that would be used in multiple classes and multiple functions throughout the program. By including this abstract class, I always knew where my more utility functions were and was able to quickly modify them during the debugging process. This saved me hours of hunting for time conversion errors especially.

The program uses SQL to access a MYSQL database. I created a helper package that contained several abstract classes to control methods for querying the database without needing to instantiate an object. These abstract classes allowed me to keep my code organized and reusable throughout the project by grouping the queries based on the tables of the database.

All user logins are monitored and stored in a .txt file. When the user presses the login button an action even is monitored and writes the username, timestamp, location, and if the login was successful. This was all accomplished with various functions from the Java IO API.

The Home Screen of the app contains several buttons that direct the user to new windows (i.e., adding, updating, and creating reports). This functionality duplicated many lines of code and cluttered the controller for the Home Screen. To clean the code up, I used a lambda expression that accessed a functional interface. The functional interface had an abstract method that took in variables for a path, title, and action event. I used a lambda expression to pass the variables into the statements calling a new JavaFX scene. Now each time I called a new scene I used one line of code instead of six, cleaning up my code.

The most difficult part of this project for me was keeping the date and time conversion straight. The program needed to display all dates and times in the time zone of the local machine, while all the time in the database needed to be in UTC. To keep things simple, I created “helper” classes that I could call throughout the program that took in a date, time, and time zone and converted it to either UTC or the local time zone depending on the action being taken.

I learned the most about problem-solving and logical thinking during the exception control implementations. Some examples were preventing overlapping appointments for customers and preventing appointments outside of business hours in a different time zone than the local user. This is where I learned the value of brainstorming and pseudocode. Before starting to write code, I sat down with a whiteboard and planned exactly what the program needs to look for and some test cases that could validate the logic. This gave me a clear path forward and helped me stay anchored while implementing my solution.
